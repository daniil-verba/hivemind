# Hivemind

> **Единая децентрализованная P2P-сеть для приложений нового поколения.**
>
> Не библиотека. Не фреймворк. **Сеть, которая растёт с каждым приложением.**

---

## 🔥 Что такое Hivemind?

Hivemind — это **единая P2P-сеть**, поверх которой работают приложения. Каждый новый пользователь любого приложения усиливает сеть для всех остальных.

```
┌─────────────────────────────────────────────────────────────┐
│                      HIVEMIND NETWORK                        │
│           Единая децентрализованная P2P-сеть                │
│    Каждый узел = хранилище + маршрутизатор + ретранслятор   │
└─────────────────────────────────────────────────────────────┘
                              │
            ┌─────────────────┼─────────────────┐
            │                 │                 │
            ▼                 ▼                 ▼
       ┌─────────┐      ┌─────────┐      ┌─────────┐
       │Platform │      │RPG Game │      │Website  │
       │(чат)    │      │(чат)    │      │(поддержка)│
       └─────────┘      └─────────┘      └─────────┘
            │                 │                 │
            └─────────────────┴─────────────────┘
                              │
                    Все используют ОДНУ сеть
            Больше приложений → сильнее сеть → лучше всем
```

### Почему это важно?

| Традиционный подход | Hivemind |
|---------------------|----------|
| Каждое приложение строит свою изолированную сеть | Все приложения — одна сеть |
| Пустое приложение = пустая сеть | Пустое приложение = доступ к миллионам узлов |
| Центральные серверы = точка отказа и контроля | Нет серверов = нет блокировок, слежки |

## 🎯 Для кого Hivemind?

### Разработчикам игр
- Встроить чат, лидерборды, мультиплеер без серверов и без Firebase.
- Работает в РФ без VPN и иностранных карт.

### Разработчикам сайтов и приложений
- Реалтайм-синхронизация, уведомления, чат поддержки.
- Не нужен backend — сеть сама хранит и доставляет данные.

### Создателям контента
- Динамические P2P-страницы и сайты, неубиваемые цензурой.
- Контент живёт, пока жив хотя бы один читатель.

### Пользователям
- Общение без центральных серверов.
- Ни одна компания не контролирует ваши данные.

## 📦 Быстрый старт

### Установка

```bash
git clone https://github.com/daniil-verba/hivemind.git
cd hivemind
mkdir build && cd build
cmake ..
make -j$(nproc)
sudo make install
```

### Использование в вашем проекте

```cmake
find_package(hivemind REQUIRED)
target_link_libraries(your_app PRIVATE hivemind::hivemind)
```

### Пример: подключение к сети

```cpp
#include <hivemind/hivemind.h>

int main() {
    // 1. Создаём узел и подключаемся к сети
    Hivemind* h = hivemind_create();
    hivemind_start(h, 9999);

    // 2. Регистрируем имя в DHT
    hivemind_register_name(h, "Alice");

    // 3. Получаем сообщения из сети
    hivemind_set_message_callback(h, [](const char* from, const char* msg) {
        printf("[%s]: %s\n", from, msg);
    });

    // 4. Отправляем сообщение любому узлу сети
    hivemind_send_message(h, "Bob", "Hello from Hivemind!");

    // Сеть работает, пока работает узел
    sleep(3600);
    hivemind_destroy(h);
    return 0;
}
```

## 🤝 Присоединиться к сети

Hivemind — это **не библиотека для построения своей сети**. Это **готовая сеть**, к которой вы подключаете своё приложение.

### Преимущества для вашего приложения:

- **Мгновенная аудитория** — все пользователи Platform и других приложений уже в сети
- **Нулевая стоимость инфраструктуры** — нет серверов, нет хостинга
- **Работа в РФ** — не нужны иностранные сервисы и карты

## 🛡️ Лицензия: AGPL v3

Hivemind распространяется под **AGPL v3**. Это защищает сеть:

- ✅ Любой может использовать бесплатно
- ✅ Любой может модифицировать и распространять
- ⚠️ Если вы запускаете узел и позволяете другим подключаться — открывайте код изменений
- 💰 **Коммерческая лицензия** доступна для закрытых проектов — daniilverba123@gmail.com

**Почему AGPL?** Чтобы сеть оставалась единой. Никто не может взять код, закрыть его, создать форк — и разрушить network effects.
